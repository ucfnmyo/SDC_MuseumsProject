=======

The Metropolitan Museum of Art (MET), of New York City, collections contain artwork from all around the world spanning over 5,000 years. It is the third most visited art museum in the world, having 6,953,927 annual visitors, as well as being the largest art museum in the United States. Its permanent collection contains over two million works of art.

The Dataset

The MET’s Open Access program allows users to view the collection digitally. The museum’s dataset presents over 490,000 art objects with 44 attribute fields describing the artwork. The data is publicly available through the Metropolitan Museum Open Access Github repo. Documentation for the API that serves this data to the site is available here. Images are provided through the Met Museum Collection API.

Project aims

An ineractive tool allowing the general public to engage with the Met collection. Using data mining techniques, this explores an alternative taxonomy of artworks, challenging traditional concepts of how we interact with art by taking a look at the materials used in MET collection, and exploring how a material and method used in one country has influenced artwork made by similar materials and methods in other countries throughout history.

Data Cleaning

The first challenge was to have as many artworks associated to the country it was made in. Before cleaning the data, the MET dataset ‘country’ column contained inconsistent values (only ~76,000 objects had a country filled in the field) and the rest were null. In order to spatially visualize the collection, the ‘country’ column values must match with a country geojson values. The first step was to remove any string characters and any key fields that were not associated to the name of a country. For the remaining artworks with null values, we filled this in using the country associated to the artist’s nationality, artist’s display bio, or the culture (ryanabest). This resulted in dropping slightly more than 10% of the total objects to spatially visualize ~430,000 art objects. The next challenge was to associate each art object to the method and materials used to make the object. This involved refining the ‘classification’ column, which contained a term describing the artwork type (i.e. sculpture, painting) from 1,144 classifications into a generalized list of 205 classifications. To send the data in the correct format to MySQL, blank values were replaced with NaNs from numpy, which corresponds as null in MySQL.

Transformation

With a technique known as one-hot encoding we vectorized materials, methods and classifications to tranform the dataset into a machine-readable format, enabling advanced data analytics.

Back End

The cleaned data is stored in several tables in a MYSQL database, accessible through a node.js API. The key endpoints deliver a summary or a subset of the data according to parameters passed with a GET request. For several of the visualizations, custom endpoints were built to deliver exactly the data necessary, reducing load times and the complexity required of the code receiving the data. Documentation for the general API endpoints can be found here.

Artwork Classification

=======

Natural language processing (NLP) was implemented to obtain material and technique keywords from artwork medium data. Keyword extraction vectorized the data according to the material or technique employed in the artwork.


Understanding how certain materials and techniques were employed within artworks throughout time and space was the primary interest of this analysis. A challenge was that medium data contained inconsistent formatting and concatenated strings that prevented sufficient categorical groupings. To illustrate, a medium description can be “Belleek porcelain, with overglaze enamel decoration and gilding”. In this situation, desired materials would be ‘porcelain’ and ‘enamel’ (nouns), while techniques would involve be ‘overglaze’ and ‘gilding’ (verbs).

Artwork Classification

The classification field contained a general term describing the artwork type (e.g. paintings, sculptures). Our challenge was a lack of generalised categories covering a wide range of artworks, to illustrate, where artworks were classified as ‘Membranophone-single-headed / frame drum’ and ‘Membranophone-single-headed / kettle drum’ should be agglomerated into one category: ‘membranophone’. Our approach in achieving this involved POS tagging with TextBlob, a Natural Language Processing Library, to extract noun phrases from the text data, these fed into a keyword extraction function by flashtext, which enabled us to extract relevant artwork categories from the data. Resultanty, we refined 1,114 classifications into 205 artwork categories. With such a result we can group similar artworks in a comprehensive and efficient manner.

In order to get round this we decided to convert the coordinates to a projection, which allows us to make use of the standard DBSCAN distance function. We used the Lambart Conformal Conic projection.
